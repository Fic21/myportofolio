IoT Project

Iot Project ini adalah sebuah tugas akhir dari perkuliahan saya yang mana project ini diangkat dengan judul yang beberapa kali bertukar-tukar karena dosen pembimbing yang pada akhirnya ditetapkan dengan judul "Sistem Peringatan Keamanan Dengan GPS dan Pemutus Pengapian Jarak Jauh pada Kendaraan Bermotor" . Pada saat pengangkatan judul saya hanya bermodal nekat saja padahal waktu yang diberikan dalam penyelesaian Tugas Akhir hingga akhir September 2019 yang lebih parahnya saya tidak mengerti sama sekali bagaimana cara komunikasi data antara hardware dengan Web pada saat itu.

Bentuk Seluruh Bagian alat

Pada akhir bulan Juni 2019 tanpa berpikir panjang saya memulai mempelajari bagaimana membangun Web sederhana menggunakan HTML,PHP,dan CSS setelah seharian penuh mempelajarinya saya berhasil membangun WEB lalu hari berikutnya saya mencoba menghosting melalui hostingan gratisan yaitu infinityfree.net. Setelah itu kendala terbesar saya pada saat itu adalah "Bagaimana cara menerima dan mengirim data dari web server ke alat saya" setelah beberapa kali percobaan pengiriman data lokasi ke internet saya tidak berhasil lebih dari lima hari padahal saya sudah yakin algoritma pemograman mikrokontroler dan algoritma pemograman PHP pada web sudah benar dan saya juga telah berhasil melakukan pengiriman data melalui http request pada web browser akan tetapi tetap saja tidak berhasil server menerima data dari alat saya, di sana saya tidak menyerah saya tetap mencari alasan kenapa hal tersebut tidak berhasil. Dengan penuh semangat pada akhirnya saya juga membutuhkan seseorang memecahkan masalah saya kenapa saya tidak berhasil dalam pengiriman data tersebut. Saya akhirnya menghubungi teman saya yang kuliah pada jurusan Teknologi Informasi, dia telah mendengarkan penjelasan saya tentang alat yang saya buat namun dia tidak dapat memecahkan permasalahan saya lalu dia mempertemukan saya dengan temannya yang menurutnya dia yang paling terbaik di angkatannya, setelah bertemu dengan temannya saya lalu menjelaskan semua permasalahan saya pada akhirnya dia bersedia membantu saya. Setelah dia sibuk memecahkan masalah saya "bagaimana menerima data dari alat saya pada web" sekian lama dia utak-atik web saya tetap tidak bisa pada akhirnya saya bertanya mengenai WebServis yaitu Rest API lalu dia menyarankan saya untuk membuat rest-API.Setelah keesokan harinya saya menemui dia kembali karena saya tidak berhasil membuat rest-API karena pada saat itu aplikasi Postman tidak bisa berjalan pada laptop saya untuk testing, pada akhirnya dia besedia membantu membuatkan rest-API sederhana untuk alat saya. Setelah pembuatan Rest-API saya langsung mencoba agar WebServer bisa menerima data melalui Web Service ini tetapi tetap saja tidak berhasil. Saya tetap berusaha untuk memecahkan permasalahan tersebut pada akhirnya saya menemukan inti permasalahan yaitu pihak jasa hostingan tersebut tidak menerima HTTP Request selain dari Web Browser dengan alasan keamanan dan tentunya karena jasa gratisan juga. Disaat itu saya berusaha mencari jasa hostingan yang bisa menerima data dari alat saya pada akhirnya saya menemukan jasa hostingan awardspace.net sehingga saya menggunakan layanan hosting tersebut dengan nama domain http://iotfik.scienceontheweb.net.

Akhirnya saya berhasil menerima data dari alat saya sehingga bisa disimpan dan ditampilkan pada web yang saya buat. ketika semuanya berhasil saya lalu memutuskan untuk menggunakan Web Service Rest-API untuk pengiriman data saya karena saya menilai pengiriman data dapat dikirimkan dalam satu paket format JSON sehingga lebih memudah dalam pengiriman data dari alat saya ke WEB server tanpa berhubungan langsung ke Database. Agar benar-benar alat saya dapat mengirimkan dan menerima data dari Web Server saya menghabiskan waktu hampir satu bulanan.

Dengan berbagai proses pembuatan sistem yang saya buat saya menghabiskan waktu 2 bulan lebih untuk seluruh proses mulai dari pembuatan pemograman Web,Rancangan Rangkaian Alat, Desain Alat, dan Pemograman Alat yang telah dibuat. Dalam proses pembuatan tentunya saya tidak hanya berfokus 2 bulanan untuk menyelesaikan Tugas Akhir tersebut, saya juga menghabiskan waktu untuk menolong teman-teman satu angkatan yang meminta tolong membantu dalam pemecahan masalah pada Tugas Akhir mereka dan membantu merancang serta membantu memogram alat mereka sampai-sampai pembuatan laporan Tugas Akhir jadi agak lama ditambah dosen pembimbing yang susah sekali untuk konsultasi laporan. Akan tetapi dengan menolong teman pada saat itu saya banyak mendapatkan pengalaman baru disana sehingga saya tidak pernah merasa rugi karena hal itu.

Cara Kerja Alat

Alat ini bekerja sebagai penerima data lokasi dan pengontrolan relay untuk pemutus pengapian kendaraan bermotor. Lokasi yang diterima oleh GPS receiver akan dikirimkan langsung ke web service dengan menggunakan SIM800L melalui jaringan 2G. Ketika Web Service berhasil menerima data dan mengirimkannya ke Web Server lalu Web Service akan mengirim pesan respon yang mana pesan respon tersebut berisi keadaan perintah terakhir yang ditekan melalui button web seperti gambar di bawah.

Setelah SIM800L menerima perintah maka Mikrokontroler akan mengecek respon yang dikirimkan oleh SIM800L apakah data ON atau OFF sehingga mikrokontroler dapat melakukan tindakan ke relay yang terhubung langsung ke sistem pengapian kendaraan bermotor.

Selain menggunakan jaringan 2G pada alat ini juga menggunakan jaringan WiFi tetapi hanya bertindak sebagai remot untuk mengendalikan relay dengan bantuan aplikasi yang saya namakan aplikasi Remoto.Apk seperti gambar di bawah.
